Tutorial five - Text inputs remembering previously entered data

At the moment when a user then clicks on the change links nothing will happen. First, change the hrefs on the check answers page so they redirect the user back to the corresponding question page. This is the same set of actions from the linking pages section earlier.

Once the hrefs have been altered and are linking back to the correct pages, navigate back to the corresponding question pages.

On the first question page where users are asked to enter their name, find the code relating to the questions input. Within the input place value="{ { data[' currentname '] } }" and this shall replay whatever the user enters when they navigate back to this page

Though when the user clicks on the save and continue button, the prototype shall move the user to the second question page. This unfortunately is not expected behaviour, the user ideally would be taken back to the check answers page. This is due to the fact the user has come from the check answers looking only to edit one piece of information.

To acheive this we must add some further code to the form action on the first question page. Firstly, place a If statement above the existing form action.

For example { % if data['came-from-check-answers'] == "true" % } . Then copy the existing form action so that there is two and separate them onto different lines. Change the newly copied form action to say /prototype-kit-training-page-4?came-from-check-answers=false&.

Place the following { % else % } between the two form action lines. To finish add { % endif % } under the bottom form action. It should look like the video below.
